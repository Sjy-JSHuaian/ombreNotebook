第一章：Java简介与环境搭建 入门
  • Java是什么：1995年Sun公司推出，詹姆斯·高斯林创造。应用领域：大数据、游戏、企业服务、移动端、AI智能化。
  • JDK下载：访问 oracle.com → Products → Java → 选择LTS版本（JDK8/11/17/21/25）
  • 环境变量配置：高级系统设置 → 环境变量 → Path → 添加JDK的bin目录路径
  • 开发工具：IntelliJ IDEA（www.jetbrains.com.cn）
  • 验证安装：CMD中输入 java -version

第二章：注释与关键字 基础
  • 单行注释：// 注释内容 —— 快捷键 Ctrl+/
  • 多行注释：/* 注释内容 */ —— 快捷键 Ctrl+Shift+/
  • 文档注释：/** 文档注释 */
  • 关键字：被Java赋予特殊含义的单词，全部小写，如 public、class、static、void
// 这是单行注释
/* 这是多行注释 */
public class HelloWorld2 {
    public static void main(String[] args) {
        System.out.println("Hello World");
    }
}
第三章：变量与数据类型 核心
  • 变量定义：数据类型 变量名 = 数据值; 如 int a = 10;
  • 基本数据类型：byte、short、int、long、float、double、char、boolean
  • 类型大小顺序：double > float > long > int > short > byte
  • 命名规范：小驼峰命名法，如 userName、studentAge
  • 键盘录入：使用 Scanner 类接收用户输入
数据类型大小示例
byte1字节byte b = 127;
short2字节short s = 32767;
int4字节int i = 200;
long8字节long l = 100L; (需加L)
float4字节float f = 11F; (需加F)
double8字节double d = 3.14;
char2字节char c = '中';
boolean1字节boolean flag = true;
// Scanner 键盘录入示例
import java.util.Scanner;

Scanner sc = new Scanner(System.in);
int num = sc.nextInt();      // 接收整数
double d = sc.nextDouble();  // 接收小数
String str = sc.next();        // 接收字符串
第四章：循环与数组 重要
  • break：结束整个循环，跳出循环体
  • continue：结束本次循环，继续下一次循环
  • 数组静态初始化：int[] arr = {1, 2, 3, 4}; 或 int[] arr = new int[]{1,2,3};
  • 数组动态初始化：int[] arr = new int[5]; (指定长度，系统赋默认值)
  • 数组特点：长度一旦定义不可改变；索引从0开始
// 数组遍历
int[] arr = {1, 2, 3, 4, 5};
for (int i = 0; i < arr.length; i++) {
    System.out.println(arr[i]);
}

// 增强for循环
for (int num : arr) {
    System.out.println(num);
}
⚠️ 常见问题：数组索引越界 ArrayIndexOutOfBoundsException —— 确保索引在 0 到 length-1 范围内。
第五章：方法与重载 核心
  • 方法定义格式：public static 返回值类型 方法名(参数) { 方法体; return 返回值; }
  • 方法重载：同一个类中，方法名相同，参数列表不同（个数、类型、顺序）
  • 重载规则：与返回值类型无关，只与参数有关
  • 方法的好处：提高代码复用性和可维护性
// 方法重载示例
public class Calculator {
    public static int add(int a, int b) {
        return a + b;
    }
    public static double add(double a, double b) {
        return a + b;
    }
}
第六章：Java运行机制与内存 原理
  • 编译与运行：.java 源文件 → 编译(javac) → .class 字节码 → JVM运行
  • 跨平台原理：Write Once, Run Anywhere —— JVM虚拟机屏蔽底层差异
  • JVM内存分区：栈(方法调用)、堆(new的对象)、方法区(字节码)、本地方法栈、程序计数器
  • 基本数据类型：变量存储真实数据值
  • 引用数据类型：变量存储内存地址值
🧠 内存地址规则：
32位系统最大支持 2³² = 4GB 内存
64位系统最大支持 2⁶⁴ ≈ 17179TB
第七章：面向对象基础 核心
  • 面向对象思想：把数据和方法组织为一个整体，利用对象进行软件开发
  • 类的定义：JavaBean类（属性+行为）和测试类（带main方法）
  • 创建对象：类名 对象名 = new 类名();
  • private关键字：私有化成员变量，提供public的get/set方法访问
  • this关键字：区分成员变量和局部变量，代表当前对象
// 标准JavaBean类
public class Dog {
    private String name;
    private int age;

    public void setName(String name) {
        this.name = name;
    }
    public String getName() {
        return name;
    }
}
第八章：构造方法 重要
  • 构造方法格式：修饰符 类名(参数) { 方法体; }
  • 特点：方法名与类名相同；无返回值类型（void也不写）；创建对象时自动调用
  • 默认构造方法：如果不写，系统自动提供无参构造；如果写了任意构造方法，默认构造消失
  • 构造方法重载：无参构造 + 带参构造，两者构成重载
  • 习惯：无论是否使用，都手动写无参构造和带全部参数的构造
public class Student {
    private String name;
    private int age;

    // 无参构造
    public Student() {}

    // 带参构造
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
第九章：面向对象原理篇 进阶
  • 对象创建流程：加载class → 声明变量 → 堆中开辟空间 → 默认初始化 → 显示初始化 → 构造方法初始化 → 赋值地址
  • 方法调用：进栈执行，执行完毕出栈，局部变量消失
  • 对象传递：传递的是内存地址值，多个变量指向同一对象时会相互影响
  • 垃圾回收：没有任何引用指向的对象会被JVM回收
  • this本质：代表所在方法调用者的内存地址
📌 总结：JavaBean类标准写法
1. 私有化成员变量 (private)
2. 提供无参构造方法
3. 提供带全部参数的构造方法
4. 提供public的get/set方法
